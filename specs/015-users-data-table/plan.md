# Implementation Plan: User Management Data Table

**Branch**: `015-users-data-table` | **Date**: 2026-04-14 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/015-users-data-table/spec.md`

## Summary

Admin-only page that lists all platform users with server-driven pagination, sorting, multi-filtering, and search; supports per-row role changes, ban/unban (with reason), and bulk role/ban actions. URL-synced table state makes every view shareable. Builds on the already-delivered Phase 1 RPCs (`admin_list_users`, `admin_change_user_role`, `admin_ban_user`, `admin_unban_user`) and Phase 3 reusable `components/data-table/*` primitives. Follows module-first architecture: a new `modules/admin-users/` module owns the page, columns, hooks, dialogs, and server action wrappers; the route file under `app/[locale]/dashboard/users/` only wires URL to the page.

## Technical Context

**Language/Version**: TypeScript 5.x + Next.js 16 (App Router), React 19
**Primary Dependencies**: `@tanstack/react-table` ^8.21.3, `nuqs` ^2.8.8 (URL state), `next-intl` ^4.7.0, `sonner` ^2.0.7, `date-fns` ^4.1.0, `lucide-react` ^0.562.0, `react-hook-form` ^7.69.0 + `zod` ^4.2.1 (ban/role dialogs), shadcn/ui (radix primitives), Tailwind CSS ^4
**Storage**: Consumes existing Supabase RPCs `admin_list_users`, `admin_change_user_role`, `admin_ban_user`, `admin_unban_user` via `lib/supabase/server.ts`. No new DB work in this feature.
**Testing**: `npm run check` (lint + type-check + format); manual verification per `quickstart.md`
**Target Platform**: Desktop-first admin web UI served via Next.js (modern evergreen browsers); responsive to tablet/mobile via horizontal scroll
**Project Type**: Web application (Next.js App Router, module-first)
**Performance Goals**: LCP < 2.5s, CLS < 0.1; filter/sort/search round-trip < 1s p95 on 100k users (SC-003); bulk action on ≤ 100 rows < 15s (SC-005)
**Constraints**: Server-first rendering with an isolated client data-table boundary; server-driven pagination/sort/filter/search only (no in-memory cross-page ops); URL is the single source of truth for table state; admins cannot self-ban/self-role-change
**Scale/Scope**: 1 page, ~8 columns, 1 module, ~12–15 new files; expected admin audience in the low dozens; user table up to 100k rows
